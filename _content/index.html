GEant Monte-Carlo is a program based on geant4 to simulate the passage of particles through matter. It provides:

  • Database for geometry and materials
  • Emulation of hardware electronics
  • Plugins mechanism for generators, fields, digitization, (in)efficiencies, output

Code-independent experiment description

GEMC uses geometry stored in databases (for example mysql). In addition:

  • Complex geometries can be created without programming knowledge
  • Models can be imported from CAD and/or GDML
  • The geometry can be modded at run time, for example by applying tilts / displacements




A system's geometries and materials are loaded from various sources into GEMC. The Geant4 world is formed, particles are swam through materials, hits are digitized, and output(s) are created.

Geometry Variations

A detector can be re-used in multiple experiments, often with changes such as a shift of some components, a change of materials, the addition or removal of certain volumes.

In GEMC this is controlled by a string called variation. This has the advantages:

  • Multiple versions of the geometry are indexed using variations
  • The desired geometry variation is selected by using the corresponding string

In the left animation two variations of the CLAS12 Central Detector are shown. The geometries are identical except for the position of the target and are selected using one of these two JSON directives:

{ "system": "clas12CD", "variation": "nominal" }
{ "system": "clas12CD", "variation": "tshift" }
